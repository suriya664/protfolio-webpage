Alex Morgan Portfolio Template

Static HTML template featuring three landing variants, rich inner pages, and a Bootstrap-powered admin UI.

Getting started

All pages are static HTML and use CDN-hosted dependencies for Bootstrap 5, Font Awesome 6, GLightbox, Isotope, Typed.js, particles.js, and Chart.js.

  1. Open index.html in a browser to view the default landing page.
  2. Review alternative landings at index-2.html and index-3.html.
  3. To customize styling, edit files inside assets/css/ — primary tokens live in style.css.
  4. JavaScript interactions (nav, filters, typed text, lightbox, counters, admin toggles, etc.) live in assets/js/main.js.

Key dependencies

  • Bootstrap 5 via CDN (see assets/css/bootstrap.min.css & assets/js/bootstrap.bundle.min.js).
  • Font Awesome for iconography.
  • GLightbox for portfolio lightbox galleries.
  • Isotope for portfolio filtering and masonry layout.
  • Typed.js for hero typing animation with fallback in main.js.
  • particles.js for hero background effects on the main landing.
  • Chart.js for analytics visualizations in the admin UI.

Customization notes

  • Theme toggle: Works across pages via data-toggle-theme buttons and theme-switcher.js.
  • Hero variants:
    • index.html — classic hero with particles, typed text, CTA buttons, and counters.
    • index-2.html — video background hero.
    • index-3.html — minimalist split layout with theme toggle.
  • Portfolio pages: Use Isotope filtering, GLightbox modals, and support fallback filtering when JS dependencies are unavailable.
  • Admin area: Sidebar toggled via data-admin-toggle (mobile support). Each page is structured with Bootstrap cards, tables, and forms.

Forms & interactivity

  • Contact forms: Basic HTML forms with client-side validation hooks; they do not submit to a backend.
  • Counters & progress: Animated on scroll using Intersection Observers in main.js.
  • Portfolio filter fallback: If Isotope CDN fails, vanilla JS filtering keeps items usable.
